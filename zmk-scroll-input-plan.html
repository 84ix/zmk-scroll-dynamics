ZMK Scroll Dynamics Processor Design

smooth scroll OFF 前提で、axis snap、scroll acceleration、scroll inertia、remainder quantization を1つに統合する ZMK input processor の設計書。

Purpose

自作マウスのスクロール処理を、OS差の少ない通常 wheel report で実現する。ZMK の high-resolution smooth scroll は使わず、firmware側で入力を整形し、通常の wheel / horizontal wheel に変換する。

対象processor名は仮に zmk,input-processor-scroll-dynamics とする。

Requirements

  1. macOS / Windows 11 の両方で実用的なスクロール量にする。
  2. 小さいスクロールは精密に、大きいスクロールは加速と慣性で伸ばす。
  3. smooth scroll OFF でも、時間的に連続した wheel report により体感のがくつきを減らす。
  4. 縦横が混ざらないように、即時 axis snap / lock を行う。
  5. snap 判定のために入力を保留しない。初動遅延は 0ms とする。
  6. 逆方向入力が入ったら inertia を即時キャンセルする。
  7. 整数 wheel 出力に変換する際、小数相当の剰余を保持して小入力を捨てない。

Non-Goals

  • CONFIG_ZMK_POINTING_SMOOTH_SCROLLING=y の利用。
  • OSごとの標準プロファイル切り替え。
  • 縦横同時の自由な2Dスクロール。
  • ポインタ移動用 acceleration processor の完全な互換再実装。

Host Constraint

CONFIG_ZMK_POINTING_SMOOTH_SCROLLING=y は HID Resolution Multiplier ベースの smooth scrolling を使う。ホスト OS ごとに解釈が違うため、同じファームウェア設定で macOS と Windows を完全に揃えるのは難しい。

このprocessorは high-resolution wheel ではなく通常 wheel を出力する。OS間の差を完全には消せないが、smooth scroll ON時の極端な倍率差は避ける。

Kconfig

CONFIG_ZMK_POINTING=y
# CONFIG_ZMK_POINTING_SMOOTH_SCROLLING is not set

Pipeline

processorは raw pointer delta を受け取り、scroll delta を出力する。既存processorを直列に積むのではなく、内部状態を一元管理する。

raw dx/dy
  -> direction transform
  -> zero-wait axis snap
  -> acceleration curve
  -> inertia state update
  -> fixed-point accumulator
  -> integer wheel/hwheel output

Runtime State

enum locked_axis {
    AXIS_NONE,
    AXIS_X,
    AXIS_Y,
};

struct scroll_dynamics_state {
    enum locked_axis locked_axis;
    int64_t last_event_ms;
    int64_t last_input_ms;
    int32_t remainder_x;
    int32_t remainder_y;
    int32_t velocity;
    int32_t inertia_velocity;
    int32_t gesture_distance;
    uint8_t gesture_events;
    int8_t last_direction;
    bool inertia_active;
};

数値は原則として fixed-point の 1000 倍スケールで保持する。浮動小数点は使わない。

Axis Snap Algorithm

snapは入力を保留しない。最初のイベントから即時に主軸を決め、非主軸を抑制する。

if now - last_input_ms > snap_idle_ms:
    locked_axis = AXIS_NONE
    gesture_distance = 0
    gesture_events = 0

abs_x = abs(dx)
abs_y = abs(dy)

if axis_mode == "x":
    locked_axis = AXIS_X
else if axis_mode == "y":
    locked_axis = AXIS_Y
else if locked_axis == AXIS_NONE:
    if abs_x * 1000 >= abs_y * snap_ratio:
        locked_axis = AXIS_X
    else if abs_y * 1000 >= abs_x * snap_ratio:
        locked_axis = AXIS_Y
    else:
        locked_axis = abs_x >= abs_y ? AXIS_X : AXIS_Y
else:
    if locked_axis == AXIS_X and abs_y * 1000 > abs_x * snap_switch_ratio:
        locked_axis = AXIS_Y
        cancel_inertia()
    if locked_axis == AXIS_Y and abs_x * 1000 > abs_y * snap_switch_ratio:
        locked_axis = AXIS_X
        cancel_inertia()

minor-axis-scale = 0 の場合、非主軸成分は完全に捨てる。値を上げれば斜め入力を少し残せるが、今回の用途では 0 を初期値にする。

Acceleration Algorithm

acceleration は inertia の前に適用する。小入力の精密さを守るため、低速域では min-factor を使い、高速域に向かって max-factor へ線形補間する。

speed = abs(delta) * 1000 / max(dt_ms, 1)

if speed <= speed_threshold:
    factor = min_factor
else:
    t = clamp((speed - speed_threshold) * 1000 / (speed_max - speed_threshold), 0, 1000)
    if acceleration_exponent == 2:
        t = t * t / 1000
    factor = min_factor + (max_factor - min_factor) * t / 1000

scaled_delta = delta * input_scale * factor / 1000000

acceleration-exponent = 1 を標準にする。2 は高速側を強調したい場合だけ使う。

Inertia Algorithm

inertia は gesture が十分速く、十分長く、複数イベント続いた時だけ開始する。細かい補正操作では発動させない。

gesture_distance += abs(scaled_delta)
gesture_events += 1
direction = sign(scaled_delta)

if reverse_cancel and direction != 0 and direction != last_direction:
    cancel_inertia()

if speed >= inertia_start_speed
   and gesture_distance >= inertia_start_distance
   and gesture_events >= inertia_min_events:
    inertia_velocity = scaled_delta * 1000 / max(dt_ms, 1)
    inertia_active = true

on inertia tick:
    inertia_delta = inertia_velocity * inertia_tick_ms / 1000
    emit_delta(inertia_delta)
    inertia_velocity = inertia_velocity * inertia_decay / 1000
    if abs(inertia_velocity) < inertia_stop_speed:
        inertia_active = false

新しい実入力が入った場合は、その入力方向が同じなら inertia を上書き更新する。逆方向なら即キャンセルする。

Quantization

smooth scroll OFF では最終出力が整数 wheel になる。小入力を捨てないため、軸ごとに remainder を保持する。

accumulator = remainder + scaled_delta
wheel_units = accumulator / wheel_step
remainder = accumulator % wheel_step

if wheel_units != 0:
    emit wheel_units on selected output axis

wheel-step を小さくすると出力頻度が上がり、大きくすると精密だが反応が遅くなる。初期値は 1000。

Edge Cases

  • dt_ms == 0 の場合は 1 として扱う。
  • 入力が snap-idle-ms を超えて止まったら gesture 状態をリセットする。
  • axis切り替え時は inertia と remainder の非対象軸をリセットする。
  • 出力値が HID の許容範囲を超える場合は clamp する。
  • speed-max <= speed-threshold の設定は無効として build assert か初期化エラーにする。
  • wheel-step <= 0 は無効。

Test Plan

Test Expected Result
小さい連続入力 remainder に蓄積され、閾値到達後に wheel が出る。入力は捨てられない。
高速入力 max-factor 方向へ倍率が上がり、通常入力より大きい wheel が出る。
短い低速入力 inertia は発動しない。
高速flick inertia tick が開始し、inertia-decay に従って自然に停止する。
逆方向入力 inertia が即時キャンセルされ、新しい方向の入力が優先される。
斜め入力 初回イベントで主軸が決まり、非主軸は minor-axis-scale に従って抑制される。
idle後の入力 前回のaxis lockに引きずられず、新しい主軸が即時に決まる。

Tuning Guide

  1. wheel-step と input-scale で基本のスクロール量を決める。
  2. min-factor を 700 - 1000 の範囲で調整し、小さい操作の精密さを決める。
  3. max-factor を 1800 - 3000 の範囲で調整し、大きい操作の伸びを決める。
  4. speed-threshold を上げると加速開始が遅くなり、下げると早くなる。
  5. inertia-start-speed と inertia-start-distance で、誤発動しない境界を決める。
  6. inertia-decay を 820 - 920 の範囲で調整し、流れ方を決める。
  7. snap-ratio を 1200 - 1600 の範囲で調整し、軸ロックの入り方を決める。

Build Milestones

  1. binding / Kconfig / CMake を追加し、空processorがビルドできる状態にする。
  2. axis-mode と wheel / hwheel 出力だけを実装する。
  3. remainder quantizer を追加する。
  4. zero-wait snap と hysteresis lock を追加する。
  5. acceleration curve を追加する。
  6. inertia timer と reverse cancel を追加する。
  7. ログを一時的に入れて実機で tuning guide の順に詰める。
